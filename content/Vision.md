# Введение

Программный продукт разрабатывается в рамках студенческого проекта для проектного практикума.

Продукт представляет из себя программный продукт, выполняющий роль агрегатора предложений о грантах и субсидиях. Помимо сбора информации из разных источников в продукте присутствует система рекомендаций основанная на ml моделях.

Продукт должен упростить процесс поиска предложений о грантах и субсидиях, что позволит снизить количество проектов, которые не удалось реализовать по причине отсутствия финансирования.

# Проблема

Информация о грантах и субсидиях распределена по множеству источников. Студентам и молодым командам приходится искать вручную и анализировать десятки неактуальных предложений. Это ведёт к потере времени и снижает вероятность подачи заявок на финансирование.

# Целевая аудитория

Главной целевой аудиторией проекта является люди, которые занимаются студенческими проектами и стартапами, которым нужны исследовательские и образовательные гранты.

# Ценностное предложение

Программный продукт позволяет пользователям получать информацию о грантах и субсидиях, собранную из множества источников. Рекомендательная система позволит не тратить время на анализ не подходящих пользователю субсидий.

Такой подход позволит в значительной мере сократить временные и ресурсные затраты на поиск финансирования.

# Виденье проекта

Ядро проекта - языковая модель и агрегатор предложений. Клиентская часть может быть представлена в виде отдельного модуля (MVP) или быть интегрирована в образовательные платформы (онлайн проектные школы, сайты вузов, сайты фондов)

# Цели

Цели описаны в порядке итераций продукта и могут быть скорректированы:

1. ML-core и агрегатор  
    Продукт собирает и формализует информацию о грантах из n источников.  
    Продукт использует ml для построения рекомендательной системы (_**Уточнить метрики**_).
    
2. Монофункциональный модуль.  
    Разрабатывается клиентская часть (WEB-приложение или tg-bot. В зависимости от сроков).  
    Продукт позволяет авторизоваться и получить текущие предложения.
    
3. Инструмент  
    Расширяется список источников, ml улучшается. Клиентская часть модифицируется и позволяет получать не только текущие предложения, но также получать уведомления о новых грантах, добавлять гранты в избранное, улучшается интерфейс.
    

# Ограничения

Используются только открытые источники.

Подача заявок находится вне рамок проекта.

Проект реализуется собственными силами.

Вычислительные мощности ограниченны.
